# 二次开发计划文档

## 1. Context

本次二次开发以 `open-pipe-media-player` 为底座，目标是把项目演进为一个适合 OrangePi AI Pro 板端演示与简历展示的 Linux 本地媒体播放器。

当前真实状态已经不是单文件 `draft4.c` 原型阶段，而是基于以下多文件主线继续演进：

- `src/main.c`
- `src/app.c`
- `src/player.c`
- `src/pipeline.c`
- `src/ui.c`
- `include/app.h`
- `include/player.h`
- `include/pipeline.h`
- `include/ui.h`

同时，原先规划中的“阶段4：模块化重构”已经完成，当前应按下面的新阶段顺序继续推进：

1. 阶段1：原型基线与远程桌面开发基线
2. 阶段2：GUI 播放器 MVP
3. 阶段3：OrangePi 板端显示/音频验证与适配
4. 阶段4：增强功能与展示材料完善

当前阶段的重点不再是继续把功能堆回 `draft4.c`，而是基于已完成的多模块主线，继续做板端运行验证、音频输出路径适配、文档整理与演示可用性提升。

## 3. 阶段3：OrangePi 板端显示/音频验证与适配

### 3.1 阶段目标

在已完成 GUI MVP 和模块化重构的基础上，优先补齐 OrangePi 板端运行验证能力，而不是继续无边界扩展 UI 功能。重点包括：

- 完成板端图形运行方式整理
- 完成 HDMI 本地显示验证
- 增加音频输出配置入口与默认回退逻辑
- 为 HDMI / default / ALSA 指定设备输出验证提供调试抓手
- 把板端验证命令与注意事项同步到项目文档

### 3.2 当前阶段3最小实现范围

本阶段优先做以下工作：

1. **板端运行入口整理**
   - 明确“桌面终端直接运行”与“SSH 绑定 `DISPLAY/XAUTHORITY` 运行”两种方式
   - 记录必要命令、环境变量与排查方法

2. **HDMI 本地显示验证**
   - 在外接显示器环境下验证窗口、视频区、按钮区、进度条、状态标签是否正常显示
   - 记录 `gtksink` / `gtkglsink` / `glsinkbin` 的实际表现

3. **音频输出配置入口**
   - 保持默认 `autoaudiosink` 可用
   - 增加可通过配置切换到其它 audio sink 的入口
   - 若指定 sink 创建失败，则回退到默认输出并打印明确日志
   - 若 sink 支持 `device` 属性，则允许传入指定设备

4. **板端音频路径验证准备**
   - 补充 `aplay -l`、`aplay -L`、`pactl info`、`fuser -v /dev/snd/*` 等调试命令
   - 为后续 HDMI / 耳机口 / 指定 ALSA 设备验证建立统一记录方式

### 3.3 本阶段暂不扩大的范围

- 暂不新增播放列表、目录扫描、上一首/下一首等增强功能
- 暂不重写整体线程模型
- 暂不把当前播放器重构成新架构，只修会阻碍板端验证的问题

## 4. 阶段4：增强功能与展示材料完善

## 7. 主要风险与处理策略

1. **HDMI / GTK 视频显示链路在板端表现不一致**
   - 保留 `gtksink` / `gtkglsink` / `glsinkbin` 的验证记录
   - 区分“启动期警告”与“真实无法显示”两类问题

2. **指定音频输出设备失败**
   - 默认优先保留 `autoaudiosink`
   - 指定 sink 创建失败时自动回退
   - 指定 `device` 不支持时打印清晰日志

3. **PulseAudio 占用 ALSA 硬件设备**
   - 先记录占用情况，再决定是否进入独占设备验证
   - 不把 `plughw` 失败误判为播放器逻辑错误

4. **阶段边界失控**
   - 阶段3只解决板端验证直接相关问题
   - 增强功能统一延后到阶段4

## 8. 验证方案

### 8.1 工程验证

- `cmake -S . -B build`
- `cmake --build build -j$(nproc)`
- 默认路径构建通过
- 如涉及 `HWACC`，补做 `ON/OFF` 构建与回退验证

### 8.2 远程桌面验证

- 在桌面终端直接运行播放器
- 在 SSH 中使用 `DISPLAY=:0` 与正确的 `XAUTHORITY` 运行播放器
- 验证 `Open / Play / Pause / Stop / Seek / 时间标签 / 状态标签`

### 8.3 HDMI 本地显示验证

- 接入 HDMI 外接显示器
- 验证窗口、视频区域、按钮区、进度条、状态标签显示是否正常
- 记录图形链路日志与实际表现
- 重点关注程序实际使用的是 `gtkglsink + glsinkbin` 还是 `gtksink` 回退路径
- 若出现 sink 创建失败或 widget 获取失败，核对终端日志中的回退提示

### 8.4 音频输出验证

- 先验证 `autoaudiosink`
- 再验证 `alsasink device=default`
- 如进入指定设备测试，记录失败原因与回退日志

### 8.5 阶段3运行与验证命令

#### 1）构建

```bash
cmake -S . -B build
cmake --build build -j$(nproc)
```

#### 2）桌面终端直接运行

```bash
./build/open-pipe-media-player
```

#### 3）SSH 复用桌面会话运行

```bash
env DISPLAY=:0 XAUTHORITY=/var/run/lightdm/root/:0 \
  ./build/open-pipe-media-player
```

#### 4）验证默认音频输出

```bash
env DISPLAY=:0 XAUTHORITY=/var/run/lightdm/root/:0 \
  ./build/open-pipe-media-player
```

默认情况下程序使用 `autoaudiosink`。

#### 5）验证 ALSA 默认设备输出

```bash
env DISPLAY=:0 XAUTHORITY=/var/run/lightdm/root/:0 \
  OPMP_AUDIO_SINK=alsasink \
  OPMP_AUDIO_DEVICE=default \
  ./build/open-pipe-media-player
```

#### 6）排查音频设备与占用

```bash
cat /proc/asound/cards
aplay -l
aplay -L
pactl info
fuser -v /dev/snd/*
lsmod | rg 'snd|soc|asoc|hdmi'
```

#### 7）图形链路快速验证

```bash
env DISPLAY=:0 XAUTHORITY=/var/run/lightdm/root/:0 \
  gst-launch-1.0 videotestsrc num-buffers=60 ! videoconvert ! gtksink
```

#### 8）阶段3联调检查项

- 能否正常打开窗口
- `Open / Play / Pause / Stop / Seek` 是否可操作
- 时间标签和状态标签是否刷新
- 视频区域是否正常显示
- 默认输出是否可出声
- 切到 `alsasink device=default` 后是否仍可出声
- 若指定设备失败，终端是否打印回退或忽略日志

### 8.6 HDMI 本地显示验证清单

接上 HDMI 外接显示器后，建议按下面顺序逐项确认：

1. **显示环境确认**
   - HDMI 显示器已点亮
   - 桌面会话已切到 HDMI 所在屏幕或可在 HDMI 上看到桌面
   - `DISPLAY=:0` 对应的仍是当前图形桌面会话

2. **程序启动确认**
   - 能正常启动播放器窗口
   - 窗口不会一闪而退
   - 终端没有出现 GTK 无法打开显示的报错

3. **视频区域确认**
   - 视频区域控件正常显示
   - 打开媒体后画面能够正常出现
   - 没有持续黑屏、花屏、严重闪烁、明显比例异常

4. **显示链路确认**
   - 终端日志中确认当前实际使用的是：
     - `Using gtkglsink + glsinkbin video output.`
     - 或 `Using gtksink video output.`
   - 若发生回退，记录具体回退原因

5. **基础交互确认**
   - `Open / Play / Pause / Stop` 可正常操作
   - 进度条可以随播放推进刷新
   - 拖动进度条后可继续播放
   - 时间标签、状态标签能同步更新

6. **异常现象记录**
   - 若出现 `gst_video_center_rect` 等告警，记录其是否仅为启动期告警
   - 若只有 HDMI 环境下出现问题，记录远程桌面与 HDMI 的行为差异

7. **阶段结论记录**
   - HDMI 下是否可正常显示窗口
   - HDMI 下是否可正常显示视频
   - 当前使用的是 GL 路径还是 gtksink 回退路径
   - 是否已具备继续做 HDMI 音频验证的前置条件
