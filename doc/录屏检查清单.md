# 录屏检查清单

## 一、录屏前检查

- [ ] 已完成项目构建
- [ ] 测试媒体文件可正常打开
- [ ] 测试目录中至少包含 2~3 个媒体文件
- [ ] 若走 SSH 演示，已确认 `DISPLAY=:0` 与 `XAUTHORITY`
- [ ] 终端字号和窗口大小适合录屏
- [ ] 播放器窗口初始大小合适，视频区和控制区无遮挡

## 二、功能检查

- [ ] `Open` 可正常打开单个媒体文件
- [ ] 媒体信息面板能显示文件名、状态、总时长
- [ ] `Play` 正常播放
- [ ] `Pause` 正常暂停
- [ ] `Stop` 后状态和进度正确复位
- [ ] 进度条会随播放推进
- [ ] 拖动进度条后能正确跳转
- [ ] `Open Dir` 可正常加载目录
- [ ] `Prev` / `Next` 可切换媒体
- [ ] 播放结束后能自动切到下一项
- [ ] 最后一项结束后停在“播放结束”

## 三、展示质量检查

- [ ] 状态标签文字清晰可见
- [ ] 时间标签清晰可见
- [ ] 媒体信息区域无遮挡
- [ ] 鼠标操作轨迹清楚
- [ ] 点击节奏不要太快，便于面试官看懂
- [ ] 终端中如展示命令，输出内容不杂乱

## 四、可选加分项

- [ ] 额外展示一次目录扫描排序效果
- [ ] 额外展示一次 seek 跳转
- [ ] 额外展示 `OPMP_AUDIO_SINK` 配置方式
- [ ] 额外展示 OrangePi / HDMI 运行环境

## 五、录屏结束前确认

- [ ] 有完整启动过程
- [ ] 有至少一次单文件演示
- [ ] 有至少一次目录播放演示
- [ ] 有上一首 / 下一首演示
- [ ] 有总结说明，能落到“Linux C / GTK / GStreamer / OrangePi 调试”这些关键词
