# 演示脚本

## 演示目标

在 1~2 分钟内展示这是一个可运行、可继续扩展、适合 OrangePi 板端演示的 Linux 本地媒体播放器。

## 演示前准备

- 已完成构建：`cmake -S . -B build && cmake --build build -j$(nproc)`
- 准备一个包含音频和视频文件的测试目录
- 准备一个单独的媒体文件用于单文件打开演示
- 若在 SSH 下演示，提前确认 `DISPLAY` 与 `XAUTHORITY`

## 演示流程

### 1. 启动程序

说明点：

- 项目使用 C + GTK3 + GStreamer
- 当前主线已经是多模块结构，而不是单文件原型

命令：

```bash
./build/open-pipe-media-player
```

### 2. 演示单文件打开

操作：

- 点击 `Open`
- 选择一个媒体文件

讲解点：

- 加载后会显示文件名、状态、总时长
- 当前不会立刻自动播放，便于展示控制逻辑

### 3. 演示基础控制

操作：

- 点击 `Play`
- 点击 `Pause`
- 再次点击 `Play`
- 点击 `Stop`

讲解点：

- 播放状态和界面标签会同步更新
- 停止后进度与时间显示会复位

### 4. 演示拖动跳转

操作：

- 再次播放
- 拖动进度条到中间位置

讲解点：

- seek 完成后会继续播放
- 项目对重复 seek 场景做过稳定性处理

### 5. 演示目录扫描与播放列表能力

操作：

- 点击 `Open Dir`
- 选择含多个媒体文件的目录
- 点击 `Play`
- 点击 `Next` 和 `Prev`

讲解点：

- 目录扫描会过滤常见媒体扩展名
- 播放列表会按文件名排序
- 上一首 / 下一首由 app 层统一编排

### 6. 演示播放结束自动切歌

操作：

- 播放非最后一项并等待结束，或使用较短样本快速演示

讲解点：

- EOS 后会自动切到下一项
- 最后一项结束后会停在“播放结束”

### 7. 演示可配置音频输出

可选讲解：

```bash
OPMP_AUDIO_SINK=alsasink OPMP_AUDIO_DEVICE=default ./build/open-pipe-media-player
```

讲解点：

- 默认使用 `autoaudiosink`
- 板端调试时可切换到指定 sink / device

## 结尾总结话术

可以用下面这段收尾：

> 这个项目重点不是堆很多花哨功能，而是把 Linux 本地播放器从可运行原型推进到结构清晰、可演示、可继续扩展的阶段，同时保留了 OrangePi 板端显示和音频输出验证空间。
