```bash
gst-launch-1.0 audiotestsrc num-buffers=1 ! autoaudiosink
gst-launch-1.0 audiotestsrc num-buffers=1 ! alsasink device=default
gst-launch-1.0 audiotestsrc num-buffers=1 ! alsasink device=plughw:CARD=LINKLIKE,DEV=0
```

结果：

- 当前已识别出 1 张 USB 声卡：`LINKLIKE`
- `autoaudiosink` 当前实际选择的是 `pulsesink`

- `alsasink device=plughw:CARD=LINKLIKE,DEV=0` 失败，原因是设备正被 `pulseaudio` 占用
- `fuser -v /dev/snd/*` 可看到占用进程为 `pulseaudio`

结论：

- 当前默认音频输出链路已经恢复可用，阶段 1 的“基础播放链路可验证”条件成立
- 现阶段可以确认 USB 音频设备已被系统识别，并且默认输出可用

## 4. OrangePi 与普通 Ubuntu PC 的差异记录

### 4.1 图形显示差异

- 在板端通过 SSH 登录时，终端默认没有 `DISPLAY`
- 当前 OrangePi 并不是没有桌面环境，而是 SSH 会话与图形桌面会话相互分离
- 当前阶段已经可以通过远程方式接入板端桌面环境，因此 GUI 开发与验证可以继续在桌面会话基础上推进
- 若需要在 SSH 环境中复用已有桌面会话，需要显式指定 `DISPLAY=:0` 和正确的 `XAUTHORITY`
- HDMI 本地显示属于后续阶段独立验证项，不作为当前阶段开发前置条件

### 4.2 GStreamer 视频 sink 差异

- `gtksink`、`gtkglsink` 不是最小运行时环境默认自带的，必须安装 `gstreamer1.0-gtk3`
- `glsinkbin` 需要 `gstreamer1.0-gl`
- OrangePi 板端后续若引入硬件显示链路，仍需继续验证 `HWACC` 分支和 GL/显示驱动是否完全匹配

### 4.3 音频输出差异

- 当前环境中 `autoaudiosink` 会优先走 `pulsesink`
- ALSA 默认设备 `default` 仍由 PulseAudio 接管
- 当前已经接入并识别到 USB 声卡，但 `plughw:CARD=LINKLIKE,DEV=0` 会被 PulseAudio 占用
- 因此“播放成功”不代表真实声卡、HDMI 音频或耳机口音频已经按预期独占输出
- 板端需要额外确认内核模块、设备树和实际 ALSA 设备枚举结果

## 5. 当前遗留问题与处理建议

### 问题 1：SSH 会话与图形桌面会话分离

现象：

- 当前 SSH 终端 `DISPLAY` 为空
- 但系统实际存在可用的图形桌面会话，且已经可以通过远程方式看到桌面

原因：

- 当前是 SSH 会话，不是直接运行在图形桌面会话里
- 板端桌面会话与 SSH 会话彼此独立，这属于当前环境事实，不等同于系统没有桌面环境

处理方法：

```bash
env DISPLAY=:0 XAUTHORITY=/var/run/lightdm/root/:0 <命令>
```

或直接在当前可访问的桌面环境中运行播放器。

### 问题 2：HDMI 本地显示尚未接入验证

### 问题 3：显式 ALSA 硬件设备被 PulseAudio 占用

现象：

```bash
gst-launch-1.0 audiotestsrc num-buffers=1 ! alsasink device=plughw:CARD=LINKLIKE,DEV=0
```

报错显示：

- `Device 'plughw:CARD=LINKLIKE,DEV=0' is busy`

原因判断：

- 当前 USB 声卡已经被系统识别
- 但 `pulseaudio` 正占用硬件设备节点，导致 `alsasink` 无法直接独占访问 `plughw`

处理方法：

后续进入音频输出验证阶段时，建议按下面顺序处理：

```bash
cat /proc/asound/cards
aplay -l
aplay -L
pactl info
fuser -v /dev/snd/*
lsmod | rg 'snd|soc|asoc|hdmi'
```

可选方案：

- 若只要求播放器先出声，可继续使用 `autoaudiosink` 或 `alsasink device=default`
- 若要验证“指定 ALSA 设备输出”，需要临时释放 PulseAudio 对硬件设备的占用，再测试 `plughw`
- 若后续目标转向 HDMI 音频，还需继续检查对应设备是否能被 ALSA 正确枚举
- 当前默认音频链路可用，但 HDMI/耳机口/指定 ALSA 设备输出仍属于后续验证项

### 问题 4：当前主程序未传参时会回退到网络视频

现象：

- `draft4.c` 在没有输入文件路径时，会加载默认网络样例 `BigBuckBunny`
