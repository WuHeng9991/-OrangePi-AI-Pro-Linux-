# 简历项目表述

## 一句话版本

基于 OrangePi AI Pro 的 Linux 本地媒体播放器，使用 C、GTK3、GStreamer、CMake 完成图形界面播放器二次开发，支持本地媒体播放、进度拖动、目录扫描、播放列表切换与板端音视频输出调试。

## 简历精简版（适合项目经历区）

### 基于 OrangePi AI Pro 的 Linux 本地媒体播放器

**技术栈：** C / GTK3 / GStreamer / GLib / CMake / Linux

- 基于开源原型完成本地媒体播放器二次开发，将工程整理为 `main / app / player / pipeline / ui` 多模块结构，提升可维护性与可讲解性。
- 基于 GStreamer `playbin` 实现音视频播放主链，支持播放、暂停、停止、进度显示与拖动跳转，并对重复 seek 场景进行稳定性增强。
- 实现目录扫描、播放列表、上一首 / 下一首、媒体信息展示与播放结束自动切歌，提升播放器演示完整度。
- 支持通过环境变量切换音频 sink 与 device，便于在 OrangePi 板端进行 HDMI / ALSA 音频输出调试与验证。

## 简历增强版（适合面试展开）

### 项目亮点

1. **多媒体框架实践**
   - 使用 GStreamer 处理本地音视频播放链路
   - 处理 `ERROR / EOS / STATE_CHANGED / ASYNC_DONE / CLOCK_LOST / LATENCY` 等 bus 消息

2. **Linux GUI 应用开发**
   - 使用 GTK3 构建播放器窗口、控制按钮、进度条、状态标签与媒体信息区域
   - 完成文件选择、目录选择、播放控制与进度交互回调

3. **稳定性与工程化能力**
   - 将项目从单文件原型整理为多文件结构
   - 增加播放进度 watchdog 与更清晰的错误提示
   - 使用 CMake 统一构建

4. **板端调试能力**
   - 面向 OrangePi AI Pro 环境做图形界面运行、音频输出 sink 配置与桌面会话复用验证

## 面试时可重点讲的点

- 为什么从手工管线切换到 `playbin`
- seek 稳定性问题是如何定位和修复的
- app / player / pipeline / ui 各模块如何分工
- 为什么目录扫描、播放列表逻辑放在 app 层而不是 pipeline 层
- OrangePi 板端图形与音频调试时遇到过哪些典型问题
